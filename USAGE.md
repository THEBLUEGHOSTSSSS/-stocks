# 美股量化半自动助手使用说明

## 1. 项目定位

这是一个本地运行的美股半自动分析程序。

它会做五件事：

1. 抓取公开行情、宏观、新闻、期权、异动和借券数据。
2. 输出核心指数信号、持仓执行参考和新 Alpha 候选。
3. 给出基于门槛、凯利折扣和账户预算的总建议风险敞口。
4. 对空头建议增加借券费率、可借股数和动态保证金约束。
5. 生成 Markdown 报告和 JSON，供你手动决策与留档。

它不会自动下单，所有交易执行都由你自己完成。

## 2. 当前版本的主要功能

1. 核心指数信号：SPY、QQQ、DIA、IWM。
2. 市场状态判断：结合 VIX、10 年期美债、ETF 活跃度代理、QQQ/NVDA/AAOI 状态。
3. 持仓跟踪：支持你自定义录入持仓代码、方向、股数、成本和备注。
4. 自动识别：内置代码字典会自动显示股票名称和所属板块，并可自动补备注。
5. 观望强约束：市场太弱或没有高胜率模板时，总风险敞口自动归零。
6. 空头支持：已有空仓可生成 ADD_SHORT、HOLD_SHORT、COVER 参考。
7. 借券约束：新增空单会读取借券费率和可借股数，HTB 会缩量，无法借或成本过高会直接禁掉。
8. 动态保证金：空头资金占用不再固定按 50% 折算，而是按基准保证金、借券费率和维持担保比例动态计算。
9. 期权流摘要：当前只抓 NVDA 近月期权链。
10. 异动雷达：从预设股票池中扫描放量突破和破位候选。
11. 报告输出：页面预览、下载 JSON、保存 Markdown 与 JSON 到 snapshots 目录。

## 3. 目录说明

主要文件如下：

1. app.py：Streamlit 主界面入口。
2. config.py：全局配置、核心指数、股票元数据、借券阈值和股票池。
3. data：抓数与特征计算模块。
4. models：市场状态、信号、凯利等模型。
5. portfolio/holdings.json：持仓存储文件。
6. portfolio/account.json：账户总权益、空头基准保证金和单笔仓位上限。
7. reports：报告构建与落盘逻辑。
8. snapshots：保存 Markdown 与 JSON 报告的目录。

## 6. 页面怎么用

### 6.1 左侧栏：持仓录入

左侧现在是中文表单式录入，不是旧版数据表格。

每一行包含：

1. 代码
2. 方向
3. 股数
4. 成本价
5. 备注

你可以使用：

1. 新增一行：增加新的持仓输入行。
2. 重置为已保存：丢弃当前编辑内容，恢复到 portfolio/holdings.json。
3. 保存持仓：把当前输入写回 portfolio/holdings.json。

### 6.2 代码搜索与自动识别

“代码”输入框支持两种方式：

1. 直接输入字母，使用候选列表搜索已收录代码。
2. 直接手动输入新的代码，即使它不在当前内置字典里也可以保存。

如果代码命中内置字典，页面会显示：

1. 股票名称。
2. 所属分类，例如半导体、网络安全、指数 ETF。

若备注为空，系统会自动把分类填入备注；如果你已经手工改过备注，系统不会强制覆盖。

### 6.3 左侧栏：账户资金参数

除了持仓，左侧还有一组账户资金参数。

主要包括：

1. 账户总权益。
2. 空头基准保证金折算。
3. 单笔多头目标仓位上限。
4. 单笔空头目标仓位上限。

含义要点：

1. 空头基准保证金折算不是最终实际占用，而是动态保证金模型的起点。
2. 新空单的真实资金占用，会在这个基准上叠加借券费率。
3. 若借券费率高或可借股数不足，系统会自动缩量甚至禁空。

### 6.4 凯利折扣

左侧有一个“凯利折扣”滑块，范围是 0.1 到 1.0。

含义：

1. 1.0 表示满凯利。
2. 0.5 表示半凯利，也是当前默认值。
3. 数值越小，总建议风险敞口越保守。

如果你更重视回撤控制，优先使用 0.5 或更低。

### 6.5 运行分析

点击“运行分析”后，程序会做这些事：

1. 抓取核心指数信号标的、市场状态辅助标的和你的持仓数据。
2. 计算技术指标、相对强弱、波动率和突破状态。
3. 获取新闻并构建情绪特征。
4. 拉取 VIX、10 年期美债收益率和 ETF 活跃度代理。
5. 拉取 NVDA 近月期权链摘要。
6. 扫描预设股票池的异动候选。
7. 查询空头持仓和空头候选的借券费率与可借股数。
8. 计算市场状态、门控结果、动态保证金、执行单和凯利权重。

## 7. 四个标签页怎么看

### 7.1 核心信号

这里显示的是核心指数信号，不是你的个股持仓。

当前默认展示：

1. SPY
2. QQQ
3. DIA
4. IWM

表格里会看到：

1. 推演日志。
2. 信号模式。
3. 是否允许承担风险。
4. 门控原因。
5. 目标 EV。
6. 胜率。
7. 波动惩罚。
8. 目标权重。
9. RSI14 与 ATR14。

这个标签页下方还会显示 NVDA 的期权流摘要。

### 7.2 持仓盈亏

这里显示三部分：

1. 账户资金概览：总权益、市场资金、闲余资金、多空敞口和动态空头占用。
2. 持仓市值表：包括代码、方向、股数、成本价、最新价、市值和浮盈亏。
3. 旧仓位执行单：给已有持仓的 ADD、HOLD、REDUCE、ADD_SHORT、HOLD_SHORT、COVER 或 LIQUIDATE 参考。

要点：

1. 普通多头只有在动能模板成立且通过门槛时，才可能给出 ADD。
2. 如果市场太弱，或该持仓未通过门槛，通常只会给出 HOLD。
3. 如果持仓是空头，系统会先判断空头模板，再检查借券条件，可能给出 ADD_SHORT、HOLD_SHORT 或 COVER。
4. 如果持仓是 AAOX，则会用 AAOI 的破位结果做代理，环境弱或底层破位时会给出 LIQUIDATE。

旧仓位执行单里新增的空头相关字段包括：

1. 借券费率%。
2. 可借股数。
3. 难借状态。
4. 禁空。
5. 初始保证金%。
6. 维持担保%。

### 7.3 异动雷达

这里也分两块：

1. 全市场异动雷达：展示候选股的突破、量能、预期收益、EV 和胜率等信息。
2. 新 Alpha 目标：从候选里进一步挑出可执行程度更高的对象。

如果全局已经触发观望强约束，这一页里的新目标会统一显示 IGNORE。

如果候选是空头方向，还会额外发生两层约束：

1. 借券费率大于等于 30%，或可借股数小于等于 0：直接 IGNORE。
2. 借券费率大于等于 1%：标记为 hard-to-borrow，并自动缩小建议仓位。

### 7.4 报告与 JSON

这里会显示：

1. Markdown 报告预览。
2. JSON 执行单。
3. 保存 Markdown + JSON 到 snapshots 的按钮。
4. 下载 JSON 的按钮。

报告中的 Portfolio_Budget 现在还会包含借券摘要、每个空头的保证金比例和禁空状态。

## 8. 观望强约束是什么意思

当前版本会在两种情况下强制进入空仓/观望：

1. 市场分数小于等于 0.34。
2. 没有任何核心信号、持仓信号或雷达候选通过高胜率门槛。

触发后，你会看到：

1. 页面顶部出现黄色提醒。
2. 总建议风险敞口变成 0.0%。
3. 新 Alpha 目标不会给出 BUY 或 SELL_SHORT。

这意味着系统认为当前环境不值得新增风险，而不是简单地“没抓到数据”。

## 9. 空头规则怎么看

当前版本的空头规则可以按三层理解：

1. 先过交易门槛：必须先满足 SELL_SHORT 或 ADD_SHORT 的模板。
2. 再过借券门槛：需要有可借股数，且借券费率不能过高。
3. 最后过资金门槛：系统会按动态保证金重算资金占用，预算不够时不建议开仓。

具体规则：

1. 年化借券费率大于等于 30%，直接禁空。
2. 可借股数小于等于 0，直接禁空。
3. 年化借券费率大于等于 1%，视为 hard-to-borrow，建议仓位缩小 20%。
4. 初始保证金会在你设置的空头基准保证金折算上，再叠加借券费率。

资金参数文件路径是：

1. portfolio/account.json

格式示例：

```json

  "max_long_position_pct": 0.1,
```

注意：这里的 short_margin_ratio 现在表示空头基准保证金折算，不等于最终实际空头占用比例。

## 13. 常见问题

### 13.1 页面报错 No module named streamlit

原因通常是用了错误的 Python 解释器。

解决方法：

```bash
cd '/Users/guolishuo/Desktop/美股交易 stocks'
.venv/bin/python -m streamlit run app.py --server.port 8501
```

### 13.2 页面报错 NoneType has no attribute get

这个问题曾经出现在新闻字段为空时。当前版本已经对空字段做了保护。

如果你仍然看到类似报错，通常先刷新页面再重新运行分析；若持续出现，再检查依赖或数据源返回是否异常。

### 13.3 为什么我明明有持仓，结果却显示观望

这通常不是程序坏了，而是以下情况之一：

1. 市场分数太低，触发了全局观望。
2. 你的持仓虽然有正收益预期，但没有通过 EV、胜率、盈亏比等硬阈值。
3. 当前信号属于观望模式，而不是动能延续、均值回归或空头破位模板。

### 13.4 为什么空头经常显示 IGNORE 或 HOLD_SHORT

这通常不是程序坏了，而是空头额外门槛比多头更严格。

常见原因：

1. IBorrowDesk 返回当前可借股数为 0。
2. 借券费率过高，触发禁空。
3. 空头模板本身没有通过 EV、胜率或市场环境门槛。
4. 动态保证金重算后，闲余资金不足以支撑新增空单。

### 13.5 为什么没有真实 ETF 资金净流入

因为免费数据源下，这类数据不稳定，接口口径也不统一。当前版本用 QQQ、SPY、IVV 的美元成交额相对 20 日均值做活跃度代理。

### 13.6 某些股票没有自动识别分类

说明该代码还不在当前内置元数据字典里。

这不会阻止你保存或分析该代码，只是页面不会自动补名称和分类。

### 13.7 启动时报 Port 8501 is not available

这不是代码错误，而是端口已被其他进程占用。

解决方法之一是换端口启动，例如 8511：

```bash
cd '/Users/guolishuo/Desktop/美股交易 stocks'
.venv/bin/python -m streamlit run app.py --server.port 8511
```

## 14. 当前版本边界

1. 不自动下单。
2. 新闻分析仍然是标题关键词模型，不是全文大模型解读。
3. ETF 活跃度是代理变量，不是真实资金流。
4. AAOX 仍使用 AAOI 作为底层风险代理。
5. 异动雷达是预设股票池，不是全市场实时扫描。
6. 借券可用性依赖 IBorrowDesk 当前返回值，某些时段大盘股也可能显示可借股数为 0。

## 15. 推荐使用流程

建议每天按这个顺序使用：

1. 启动页面。
2. 检查并更新左侧持仓和账户资金参数。
3. 确认凯利折扣。
4. 点击“运行分析”。
5. 先看市场状态、风险偏好分数和是否触发观望强约束。
6. 再看“持仓盈亏”里的旧仓位执行单，特别是空头借券状态和保证金占用。
7. 最后看“异动雷达”里的新 Alpha 目标。
8. 保存 Markdown 和 JSON，作为当日交易记录。
