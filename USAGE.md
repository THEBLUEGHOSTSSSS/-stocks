# 美股量化半自动助手使用说明

## 1. 适用方式

这是一个本地运行的半自动分析程序。

它会做五件事：

1. 抓取公开行情、宏观、新闻、期权、异动和借券数据。
2. 输出核心指数信号、持仓执行参考和新 Alpha 候选。
3. 给出基于门槛、凯利折扣和账户预算的总建议风险敞口。
4. 给出基于 IV、IVR 和近月 ATM 链的期权结构建议。
5. 生成 Markdown 报告和 JSON，供人工决策与留档。

它不会自动下单，所有交易执行都由你自己完成。

## 3. 左侧栏怎么填

### 3.1 持仓录入

左侧栏每一行包含：

1. 代码。
2. 方向。
3. 股数。
4. 成本价。
5. 备注。

你可以使用：

1. 新增一行：增加新的持仓输入行。
2. 重置为已保存：丢弃当前编辑内容，恢复到本地保存状态。
3. 保存持仓：把当前输入写回本地持仓文件。

### 3.2 代码识别

代码输入支持两种方式：

1. 在候选列表中搜索精选标的。
2. 直接输入新的交易代码。

系统会从三层识别标的：

1. 内置精选池。
2. 公开交易所全量代码库。
3. 用户本地新增过的代码。

首次输入的新代码会写入本地自定义代码文件，之后会出现在候选里。

如果你想手动刷新公开交易所代码库，可以执行：

```bash
cd '/Users/guolishuo/Desktop/美股交易 stocks'
.venv/bin/python -m portfolio.exchange_tickers
```

### 3.3 账户参数

左侧还需要填写：

1. 账户总权益。
2. 空头基准保证金折算。
3. 单笔多头目标仓位上限。
4. 单笔空头目标仓位上限。
5. 凯利折扣。

含义要点：

1. 凯利折扣越低，整体仓位越保守。
2. 空头资金占用不是固定 50%，而是会叠加借券费率、维持担保率等动态约束。

## 4. 页面主流程

建议按下面顺序使用页面：

1. 先录入持仓和账户参数。
2. 点击运行分析。
3. 先看核心信号和市场状态。
4. 再看持仓页里的执行参考与预算占用。
5. 再看异动雷达与新 Alpha 目标。
6. 如果要做期权，再进入期权预言机查看结构与下单步骤。
7. 最后在报告页保存 Markdown 和 JSON。

## 5. 主要页面说明

### 5.1 核心信号

这里主要看：

1. 核心指数信号：SPY、QQQ、DIA、IWM。
2. 市场状态与风险偏好分数。
3. 期权流摘要。
4. 数学增强结果。
5. 期权策略预言机。
6. 学术 Alpha 摘要。

这一页新增的数学与研究模块包括：

1. SVD 正交化因子权重。
2. PCA 残差均值回归信号。
3. 图谱套利候选摘要。
4. 学术 Alpha 主题与因子草图。

### 5.2 持仓盈亏

这里会显示：

1. 账户总权益、闲余资金、多空敞口和动态空头占用。
2. 当前持仓市值、浮动盈亏和参考执行动作。
3. 历史快照对比。
4. 成本价熔断或报价异常提示。

如果持仓是空头，系统还会额外显示：

1. 借券费率。
2. 可借股数。
3. 是否 HTB。
4. 是否禁空。
5. 初始保证金和维持担保约束。

### 5.3 异动雷达

异动雷达会并行扫描三类候选：

1. 价格与量能驱动的 breakout / breakdown。
2. 图谱套利和联动扩散候选。
3. 数学引擎产出的残差均值回归候选。

如果全局已经触发观望强约束，这一页里的新目标会被压成 IGNORE 或观察状态。

### 5.4 报告与 JSON

这里会显示：

1. Markdown 报告预览。
2. JSON 执行结构。
3. 保存 Markdown + JSON 到 snapshots 的按钮。
4. 学术 Alpha 摘要刷新入口。

## 6. 期权流与期权预言机怎么用

### 6.1 期权流摘要

期权流区域现在不再只显示“最活跃执行价”，而是优先显示：

1. 近月到期日。
2. 距到期天数。
3. 当前 ATM IV。
4. 52 周本地 IV 样本数。
5. 近月 ATM 看涨链。
6. 近月 ATM 看跌链。

ATM 链里的关键字段包括：

1. 执行价。
2. 采用权利金。
3. 权利金来源。
4. 买价 / 卖价 / 中间价。
5. 自算 IV。
6. 原始 IV。
7. 成交量与未平仓量。

### 6.2 期权预言机

在“期权策略预言机”区域，先选标的，再选策略。

页面会同步显示：

1. 当前 IV。
2. IVR。
3. 波动环境。
4. 近月到期日。
5. IV 缓存样本数。
6. 当前优先策略及其理由。
7. 到期盈亏图。
8. 香港盈立证券下单步骤。

### 6.3 当前策略选择逻辑

当前 IVR 规则是硬约束：

1. IVR > 80：只给卖波动结构，如 Short Strangle、Iron Condor。
2. IVR < 20：优先给 Long Straddle。
3. 中间区间：按持仓和风险场景在 Covered Call、Bear Put Spread、Iron Condor 等结构间切换。

### 6.4 香港盈立证券执行顺序

如果你使用香港盈立证券执行，建议按这个顺序看：

1. 先看“下单前检查”：确认账户权限、美元购买力和保证金额度。
2. 再看“具体下单步骤”：确认到期日、执行价、方向和分腿顺序。
3. 最后看“成交后管理”：看是否该止盈、平仓、滚动或减少暴露。

不要把页面策略名字当成自动下单指令。真正下单前仍要手工复核：

1. 到期日是否一致。
2. 执行价是否一致。
3. 张数是否与你的真实持仓和保证金匹配。
4. 该单是否会占用超出预期的保证金或权利金。

### 6.5 IV 缓存怎么理解

系统会把每日近月 ATM IV 写入本地缓存，并逐步形成 52 周历史序列。

因此要注意：

1. 第一天运行时，IV 样本数很少，IVR 还可能回退到代理带宽。
2. 连续运行后，本地样本会越来越多，IVR 会越来越接近真实历史分位。
3. 这个缓存属于本地运行状态，不会自动上传到仓库。

## 7. 观望强约束

系统在两种情况下会强制进入空仓 / 观望：

1. 市场分数过低。
2. 没有任何核心信号、持仓信号或雷达候选通过高胜率门槛。

触发后会出现三个结果：

1. 页面顶部出现提醒。
2. 总建议风险敞口回到 0。
3. 新 Alpha 目标不会给出激进开仓建议。

## 8. 空头规则

空头建议要同时通过三层约束：

1. 先过交易门槛。
2. 再过借券门槛。
3. 最后过资金与保证金门槛。

当前关键规则：

1. 可借股数小于等于 0，直接禁空。
2. 借券费率过高，直接禁空或显著缩仓。
3. HTB 标的即使方向看对，也会被额外压缩建议仓位。

## 9. 输出文件

点击保存后，程序会在 snapshots 目录生成两份文件：

1. quant_report_时间戳.md
2. quant_report_时间戳.json

两者来自同一份分析结果，只是展示形式不同。

## 11. 常见问题

### 11.1 8501 端口被占用

换一个端口启动：

```bash
.venv/bin/python -m streamlit run app.py --server.port 8511
```

### 11.2 页面没刷新到最新代码

如果页面内容还停留在旧文案或旧模块：

1. 先刷新浏览器标签。
2. 再确认启动的是项目目录下的 .venv。
3. 必要时重启 Streamlit 进程。

### 11.3 第一次运行 IVR 看起来不稳定

这通常不是计算错误，而是 52 周 IV 缓存样本还没积累起来。随着本地运行天数增加，IVR 会逐步转向真实历史驱动。
