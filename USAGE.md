# 美股量化半自动助手使用说明

## 1. 项目定位

这是一个本地运行的美股半自动分析程序。

它会做四件事：

1. 抓取公开行情、宏观、新闻、期权和异动数据。
2. 输出核心指数信号、持仓执行参考和新 Alpha 候选。
3. 给出基于门槛和凯利折扣的总建议风险敞口。
4. 生成 Markdown 报告和 JSON，供你手动决策与留档。

它不会自动下单，所有交易执行都由你自己完成。

## 2. 当前版本的主要功能

1. 核心指数信号：SPY、QQQ、DIA、IWM。
2. 市场状态判断：结合 VIX、10 年期美债、ETF 活跃度代理、QQQ/NVDA/AAOI 状态。
3. 持仓跟踪：支持你自定义录入持仓代码、股数、成本和备注。
4. 自动识别：内置代码字典会自动显示股票名称和所属板块，并可自动补备注。
5. 观望强约束：市场太弱或没有高胜率模板时，总风险敞口自动归零。
6. 期权流摘要：当前只抓 NVDA 近月期权链。
7. 异动雷达：从预设股票池中扫描放量突破候选。
8. 报告输出：页面预览、下载 JSON、保存 Markdown 与 JSON 到 snapshots 目录。

## 3. 目录说明

主要文件如下：

1. app.py：Streamlit 主界面入口。
2. config.py：全局配置、核心指数、股票元数据和股票池。
3. data：抓数与特征计算模块。
4. models：市场状态、信号、凯利等模型。
5. portfolio/holdings.json：持仓存储文件。
6. reports：报告构建与落盘逻辑。
7. snapshots：保存 Markdown 与 JSON 报告的目录。

## 6. 页面怎么用

### 6.1 左侧栏：持仓录入

左侧现在是中文表单式录入，不是旧版数据表格。

每一行包含：

1. 代码
2. 股数
3. 成本价
4. 备注

你可以使用：

1. 新增一行：增加新的持仓输入行。
2. 重置为已保存：丢弃当前编辑内容，恢复到 portfolio/holdings.json。
3. 保存持仓：把当前输入写回 portfolio/holdings.json。

### 6.2 代码搜索与自动识别

“代码”输入框支持两种方式：

1. 直接输入字母，使用候选列表搜索已收录代码。
2. 直接手动输入新的代码，即使它不在当前内置字典里也可以保存。

如果代码命中内置字典，页面会显示：

1. 股票名称。
2. 所属分类，例如半导体、网络安全、指数 ETF。

若备注为空，系统会自动把分类填入备注；如果你已经手工改过备注，系统不会强制覆盖。

### 6.3 凯利折扣

左侧有一个“凯利折扣”滑块，范围是 0.1 到 1.0。

含义：

1. 1.0 表示满凯利。
2. 0.5 表示半凯利，也是当前默认值。
3. 数值越小，总建议风险敞口越保守。

如果你更重视回撤控制，优先使用 0.5 或更低。

### 6.4 运行分析

点击“运行分析”后，程序会做这些事：

1. 抓取核心指数信号标的、市场状态辅助标的和你的持仓数据。
2. 计算技术指标、相对强弱、波动率和突破状态。
3. 获取新闻并构建情绪特征。
4. 拉取 VIX、10 年期美债收益率和 ETF 活跃度代理。
5. 拉取 NVDA 近月期权链摘要。
6. 扫描预设股票池的异动候选。
7. 计算市场状态、门控结果、执行单和凯利权重。

## 7. 四个标签页怎么看

### 7.1 核心信号

这里显示的是核心指数信号，不是你的个股持仓。

当前默认展示：

1. SPY
2. QQQ
3. DIA
4. IWM

表格里会看到：

1. 推演日志。
2. 信号模式。
3. 是否允许承担风险。
4. 门控原因。
5. 目标 EV。
6. 胜率。
7. 波动惩罚。
8. 目标权重。
9. RSI14 与 ATR14。

这个标签页下方还会显示 NVDA 的期权流摘要。

### 7.2 持仓盈亏

这里显示两部分：

1. 持仓市值表：包括代码、股数、成本价、最新价、市值和浮盈亏。
2. 旧仓位执行单：给已有持仓的 ADD、HOLD、REDUCE 或 LIQUIDATE 参考。

要点：

1. 普通持仓只有在动能模板成立且通过门槛时，才可能给出 ADD。
2. 如果市场太弱，或该持仓未通过门槛，通常只会给出 HOLD。
3. 如果持仓是 AAOX，则会用 AAOI 的破位结果做代理，环境弱或底层破位时会给出 LIQUIDATE。

### 7.3 异动雷达

这里也分两块：

1. 全市场异动雷达：展示候选股的突破、量能、预期收益、EV 和胜率等信息。
2. 新 Alpha 目标：从候选里进一步挑出可执行程度更高的对象。

如果全局已经触发观望强约束，这一页里的新目标会统一显示 IGNORE。

### 7.4 报告与 JSON

这里会显示：

1. Markdown 报告预览。
2. JSON 执行单。
3. 保存 Markdown + JSON 到 snapshots 的按钮。
4. 下载 JSON 的按钮。

## 8. 观望强约束是什么意思

当前版本会在两种情况下强制进入空仓/观望：

1. 市场分数小于等于 0.34。
2. 没有任何核心信号、持仓信号或雷达候选通过高胜率门槛。

触发后，你会看到：

1. 页面顶部出现黄色提醒。
2. 总建议风险敞口变成 0.0%。
3. 新 Alpha 目标不会给出 BUY。

这意味着系统认为当前环境不值得新增风险，而不是简单地“没抓到数据”。

## 9. 输出文件说明

点击保存后，程序会在 snapshots 目录生成两份文件：

1. quant_report_时间戳.json
2. quant_report_时间戳.md

两者来自同一份分析结果，只是展示形式不同。

## 10. 持仓文件说明

持仓文件路径是：

1. portfolio/holdings.json

格式示例：

```json
[
  {
    "ticker": "NVDA",
    "shares": 100,
    "cost_basis": 185.5,
    "notes": "半导体/AI算力"
  }
]
```

你可以手动编辑这个文件，但更推荐直接在左侧表单里录入并保存。

## 11. 常见问题

### 11.1 页面报错 No module named streamlit

原因通常是用了错误的 Python 解释器。

解决方法：

```bash
cd '/Users/guolishuo/Desktop/美股交易 stocks'
.venv/bin/python -m streamlit run app.py --server.port 8501
```

### 11.2 页面报错 NoneType has no attribute get

这个问题曾经出现在新闻字段为空时。当前版本已经对空字段做了保护。

如果你仍然看到类似报错，通常先刷新页面再重新运行分析；若持续出现，再检查依赖或数据源返回是否异常。

### 11.3 为什么我明明有持仓，结果却显示观望

这通常不是程序坏了，而是以下情况之一：

1. 市场分数太低，触发了全局观望。
2. 你的持仓虽然有正收益预期，但没有通过 EV、胜率、盈亏比等硬阈值。
3. 当前信号属于观望模式，而不是动能延续或均值回归模板。

### 11.4 为什么没有真实 ETF 资金净流入

因为免费数据源下，这类数据不稳定，接口口径也不统一。当前版本用 QQQ、SPY、IVV 的美元成交额相对 20 日均值做活跃度代理。

### 11.5 某些股票没有自动识别分类

说明该代码还不在当前内置元数据字典里。

这不会阻止你保存或分析该代码，只是页面不会自动补名称和分类。

## 12. 当前版本边界

1. 不自动下单。
2. 新闻分析仍然是标题关键词模型，不是全文大模型解读。
3. ETF 活跃度是代理变量，不是真实资金流。
4. AAOX 仍使用 AAOI 作为底层风险代理。
5. 异动雷达是预设股票池，不是全市场实时扫描。

## 13. 推荐使用流程

建议每天按这个顺序使用：

1. 启动页面。
2. 检查并更新左侧持仓。
3. 确认凯利折扣。
4. 点击“运行分析”。
5. 先看市场状态、风险偏好分数和是否触发观望强约束。
6. 再看“持仓盈亏”里的旧仓位执行单。
7. 最后看“异动雷达”里的新 Alpha 目标。
8. 保存 Markdown 和 JSON，作为当日交易记录。
